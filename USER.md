## Context

- 첫 인사에서 긍정적이고 따뜻한 톤.
- 짧은 간격 리마인더(예: 1분 뒤 알람) 같은 즉시 실행형 도움에 관심 있음.
- 강한 운영 규칙: 재시작/설정변경(config.apply/config.patch/update 포함)은 사전 승인 필수.
- 메인 채팅에서 진단용 exec/원문 에러 로그 노출을 매우 싫어함.
- 오류가 나면 경고만 던지지 말고 `원인/영향/다음 조치/사용자 액션`까지 완결 보고 선호.
- 외부 발송(특히 이메일)은 무조건 `최종 제목+본문`을 먼저 제시하고 사용자의 명시 승인(예: "보내") 후에만 실행.
- 발송은 보수적으로 처리: 인코딩/포맷 검증을 우선하고, 승인 전 재전송/수정 발송 금지.
- Notion 업로드는 반드시 2단계 루프: (1) 업로드 (2) 블록 타입/서식 검증. 누락 시 자동 재적용 후 재검증.
- Notion 서식 선호: 마크다운 기호(`**bold**`, `` `code` ``)를 문자 그대로 두지 말고 실제 rich_text 서식으로 변환. 코드블록/구분선/헤딩 우선.
- 모바일 가독성 선호: 긴 문단 분할, 계층 bullet 정리, 변수/심볼(`pc`, `WIDTH`, `LAST`)은 인라인 코드 칩 스타일로 강조.
- 중요 내용 강조 선호: 빨간색은 과하지 않게 사용. 기본은 볼드/구조화 중심, **경고·주의·리스크 문장에 한해** 빨간 경고 콜아웃을 선택적으로 적용.
- 공통 포맷 규칙(전 카테고리 적용): `c언어 공부 / pulse / 영어회화` 모두 동일하게 볼드·인라인코드·코드블록·구분선·계층 리스트를 적용하고, 업로드 후 서식 검증 루프를 반드시 수행.
